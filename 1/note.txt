Tipuri de atacuri cibernetice si modalitati de prevenire:

Malware - un tip de atac cibernetic rau intentionat cu virusi, troieni sau ransomware, care se poate preveni prin instalarea unui sistem antivirus actualizat regulat. De asemenea este important a nu deschide linkuri suspecte si a naviga siteuri indoielnice si a nu downloada fisiere din surse necunoascute. Ransomware se refera la acele atacuri de tip criptare sau blocaj prin care utilizatorului i se ia accesul la sistemul de operare si se cere rascumparare pentru obtinerea accesului.

Atacurile de tip phishing - sunt incercari de a pacali utilizatorul si de a-i lua date confidentiale personale sau financiare. Pentru a evita asemenea atacuri este important sa nu furnizam date confidentiale pe siteuri indoielnice sau alte forme nesigure.

Atacurile de tip DDoS implica supraincarcarea unui server sau a unei retele cu un volum mare de cereri de la mai multi utilizatori. Acest  lucru poate fi prevenit prin urmarirea traficului nedorit in retea si blocarea acestuia, deci prin aplicarea unor masuri de securitate la nivel de retea.

Atacuri de tip brute-force inseamna incercarea repetate de a ghici parole sau date criptate pentru obtinerea de acces neautorizat. Acest lucru poate fi prevenit prin alegerea de parole puternice si unice si setarea blocarii temporare a contului dupa un anumit numar de incercari.

Vulnerabilitati de securitate si erori de software - aceste metode sunt folosite de atacatori pentru a obtine acces neautorizat si poate fi prevenit prin mentinerea unor sisteme de securitate actualizate.

Ingineria sociala inseamna utilizarea manipularii sociale pentru a obtine informatii confidentiale si acces neautorizat. Prevenirea se face prin a avea grija de interactiunile social media si evitarea oferirii de informatii confidentiale.



3.Metode de securitate ale formularelor

Validarea la nivel de client: putem utiliza JavaScript pentru a valida datele inainte de a le trimite la server si astfel vom evita erorile.

Validare la nivelul serverului: odata ajunse la server, datele trebuie validate aici. In acest sens, putem folosi spre exemplu functia filter_var().

Escaparea si filtrarea datelor. Orice date introduse de utilizator si afisate pe pagina trebuie sa fie escapate si filtrate. Se poate utiliza functia htmlspecialchars() pentru a converti caracterele speciale si a preveni injectiile de cod.

Utilizarea declaratiilor pregatite. Pentru a preveni atacul de tipul injectiilor SQL putem folosi prepared statements/declaratiile pregatite in cadrul interogarilor catre baza de date, ceea ce separa datele introduse de utilizator de codul SQL si face acest tip de atac mai dificil sau imposibil.

Limitarea si validarea datelor de intrare - inainte de a valida datele de intrare, ne asiguram ca acestea au o anumita lungime utilizand de ex strlen() pentru lungimea de caractere si is_numeric() pentru a verifica daca un anume camp contine doar cifre.

Implementarea protectiei de tip Cross Site Scripting (XSS)- putem utiliza functia strip_tags() pentru a elimina etichetele din codul html introdus de utilizator pentru a preveni injectiile de script.

Adaugarea tokenurilor Cross Site Request Forgery asigura ca cererile introduse sunt autentice si nu sunt trimise de altundeva decat de pe site.

Adaugarea captchelor - ajuta la stabilirea ca persoana care introduce datele este un om si nu un program automatizat (bot), si previne atacurile automate si spamurile.
